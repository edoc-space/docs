import { Changelog } from '@edoc-space/plugin-changelog'
import { Hero, Section } from '@/Components/Mdx'

<Hero
  title="Changelog"
  eyebrow="@edoc-space/plugin-changelog"
  subtitle="React-компонент для вывода истории изменений внутри MDX-страницы."
>
  Может читать Markdown или JSON, группировать изменения по категориям и превращать номера задач в ссылки на issue tracker.
</Hero>

<Section title="Установка">
  Пакет планируется публиковать в npm под организацией `@edoc-space`.

```shell
cd local/plugins
yarn add @edoc-space/plugin-changelog
```

```shell
cd local/plugins
npm install @edoc-space/plugin-changelog
```

Пакет на npm: [@edoc-space/plugin-changelog](https://www.npmjs.com/package/@edoc-space/plugin-changelog)
</Section>

<Section title="Подключение">
Минимальный Markdown-файл:

```markdown
# Changelog

## 1.1.0 - 2026-06-30

### Added

- Добавлена публичная страница API.

### Fixed

- Исправлена навигация в документации.
```

Пример MDX-страницы:

```mdx
import { Changelog } from '@edoc-space/plugin-changelog'

<Changelog
  title="История изменений"
  source="/storage/edoc/static/changelog.md"
/>
```
</Section>

<Section title="Пример">
  Ниже компонент использует демонстрационный changelog из `static`.
</Section>

<Changelog
  title="Изменения E-Doc"
  source="/storage/edoc/static/examples/changelog.md"
  repositoryUrl="https://example.test/edoc"
/>
