# Кеш

E-Doc умеет отдавать документацию без базы данных. Но на production полезно кешировать индекс документации и уже отрендеренные документы, потому что построение навигации и разбор Markdown или MDX на каждый запрос могут стать дорогими.

## Включить кеш

Кеш документации управляется двумя переменными окружения:

```env
APP_DOCUMENTATION_CACHE=1
APP_DOCUMENTATION_CACHE_TTL=86400
```

`APP_DOCUMENTATION_CACHE=1` включает кеш документации.
Чтобы выключить кеш, укажите `0`, `false`, `off` или `no`.

`APP_DOCUMENTATION_CACHE_TTL` задает срок жизни кеша в секундах.
Значение по умолчанию - один день.
Если указать `0` или меньше, записи кеша будут сохраняться без срока истечения.

## Почему нужен prune

`phpsoftbox/cache` удаляет expired-запись файлового кеша, когда приложение снова обращается к тому же ключу. Это правильное lazy-поведение, но оно не очищает все старые файлы.

Ключи кеша документации включают fingerprint контента. Когда документ меняется, E-Doc записывает новый ключ кеша. Старый ключ может больше никогда не запрашиваться, поэтому lazy-удаление его не тронет.

Еще после прерванной записи кеша могут остаться временные файлы. Это не cache entry, и чтение ключей их тоже не удалит.

Поэтому на production нужно периодически запускать cache prune.

## Cron

Для этой задачи E-Doc не требует `phpsoftbox/scheduler`. Обычного системного cron достаточно, а базовая установка остается проще.

Для простого PHP-deploy:

```cron
17 3 * * * cd /path/to/edoc/local/backend && php psb cache:prune --tmp-max-age-hours=24 --max-age-days=30
```

Для Docker-deploy:

```bash
docker compose run --rm php-cli php psb cache:prune \
  --tmp-max-age-hours=24 \
  --max-age-days=30
```

## Что удаляет prune

`cache:prune` удаляет:

- expired-записи кеша;
- битые cache-файлы;
- старые временные cache-файлы;
- permanent-записи старше `--max-age-days`, только если эта опция передана явно.

Команда не вызывает `cache:clear`, поэтому свежий валидный кеш остается на месте.
