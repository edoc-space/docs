# Документация проекта

Раздел `/ru/docs` нужен для основной документации русской версии: введения, установки, руководств, справочников и внутренних соглашений проекта.

В отличие от отдельных страниц сайта, документация обычно растет деревом. В ней появляются разделы, вложенные документы, ссылки между страницами и порядок в левом меню.

## Минимальный пример

Начать можно с одного файла:

```text
local/storage/edoc/ru/docs/start/introduction.md
```

```markdown
---
title: Введение
description: С чего начать работу с проектом.
sidebar_position: 1
---

# Введение

Коротко объясните, что это за проект и куда идти дальше.
```

После сохранения документ откроется по адресу:

```text
/ru/docs/start/introduction
```

## Создать раздел

Когда документов становится больше, их удобнее группировать в каталогах:

```text
local/storage/edoc/ru/docs/
  start/
    index.json
    installation.md
    configuration.md
```

Файл `index.json` описывает раздел:

```json
{
  "label": "Быстрый старт",
  "description": "Первые шаги после установки проекта.",
  "position": 10,
  "collapsed": false,
  "link": { "type": "generated-index" }
}
```

Так E-Doc понимает:

- как назвать раздел в левом меню;
- где поставить его среди других пунктов;
- должен ли раздел быть свернут по умолчанию;
- нужно ли создать страницу-оглавление для каталога.

## Создать документ внутри раздела

Документ внутри каталога выглядит так же, как документ в корне:

```text
local/storage/edoc/ru/docs/start/installation.md
```

```markdown
---
title: Установка
description: Как подготовить окружение и открыть сайт локально.
sidebar_position: 1
---

# Установка

Опишите шаги установки так, чтобы читатель видел результат после каждого шага.
```

Страница откроется по адресу:

```text
/ru/docs/start/installation
```

## Порядок в меню

Для документов используется `sidebar_position`:

```yaml
sidebar_position: 20
```

Для разделов используется `position` в `index.json`:

```json
{
  "label": "Настройка",
  "position": 30
}
```

Чем меньше число, тем выше пункт в меню. Если позиция не указана, E-Doc сортирует пункты по названию.

## Описания страниц

`description` помогает читателю понять, зачем открывать страницу. Он используется в оглавлениях разделов и поиске:

```yaml
description: Как подключить OpenAPI-спецификацию к странице документации.
```

Хорошее описание отвечает на вопрос “что я здесь найду”, а не повторяет заголовок другими словами.

## Страница-оглавление раздела

Если в `index.json` указан generated index:

```json
{
  "label": "Контент",
  "link": { "type": "generated-index" }
}
```

то сам раздел становится страницей. На ней E-Doc покажет вложенные документы и их описания.

Это удобно для крупных разделов: читатель открывает раздел и видит, какие темы в нем есть.

## Ссылки между документами

В документах можно использовать обычные Markdown-ссылки:

```markdown
[Перейти к настройке](configuration.md)
```

Если страницы лежат в одном каталоге, относительная ссылка будет понятнее абсолютной. При переносе раздела такие ссылки легче поддерживать.

Можно ссылаться и на заголовок внутри документа:

```markdown
[Проверка результата](installation.md#проверка-результата)
```

## Когда нужен новый раздел

Раздел стоит создавать, если:

- внутри уже есть несколько связанных страниц;
- тема требует отдельного оглавления;
- пользователь будет искать ее как самостоятельную часть документации.

Если страница одна и не требует группировки, оставьте ее обычным документом в корне `/ru/docs`.

## Следующий шаг

Когда структура готова, можно разобраться с форматами контента: [Markdown и MDX](/ru/docs/content/markdown-and-mdx) объясняет, когда достаточно обычного текста, а когда нужны React-компоненты.
