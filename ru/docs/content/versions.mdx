# Версии документации

E-Doc хранит последнюю документацию в обычном дереве `docs`. Прошлые версии описываются в `versions.json` и могут переопределять только те файлы, которые действительно изменились.

Текущая версия всегда открывается по `/docs` или `/ru/docs` для русской локали. Поддерживаемые прошлые версии открываются по `/docs/v/{version}` или `/ru/docs/v/{version}`. Архивные версии остаются видимыми на странице "Все версии", но дерево документации и поиск для них отключены.

## versions.json

```json
{
  "current": "1.2.0",
  "versions": [
    { "version": "1.2.0", "label": "v1.2.0", "status": "current", "released_at": "2026-07-13" },
    { "version": "1.1.0", "label": "v1.1.0", "status": "supported", "released_at": "2026-07-10" },
    { "version": "1.0.0", "label": "v1.0.0", "status": "supported", "released_at": "2026-07-01" }
  ]
}
```

Статусы:

- `current` - последняя документация, доступна по `/docs` или `/ru/docs`.
- `supported` - прошлая поддерживаемая версия, доступна по `/docs/v/{version}` или `/ru/docs/v/{version}`.
- `archived` - отображается только как история; документация и поиск отключены.

Поле `released_at` опционально, но рекомендуется: оно отображает дату выпуска в списке версий и changelog.

## since и until

Документ или раздел может указать версионное окно во front matter:

```md
---
title: Новая возможность
since: 1.2.0
---
```

С `since: 1.2.0` страница не будет отображаться в `1.1.0` и более старых версиях. `until` работает наоборот и скрывает контент после указанной версии.

Если изменилась только часть существующей страницы, создайте override для старой версии, а актуальный текст оставьте в обычном дереве `docs`.
