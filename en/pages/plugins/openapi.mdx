<Hero
  title="OpenAPI"
  eyebrow="@edoc-space/plugin-openapi"
  subtitle="A React component for rendering an OpenAPI specification inside an MDX page."
>
  Suitable for public API documentation: it groups endpoints by tags, shows parameters, request bodies, responses, and short schema descriptions.
</Hero>

<Section title="Installation">
  Install the package in the user plugin layer.

```shell
cd local/plugins
yarn add @edoc-space/plugin-openapi
```

```shell
cd local/plugins
npm install @edoc-space/plugin-openapi
```

Package on npm: [@edoc-space/plugin-openapi](https://www.npmjs.com/package/@edoc-space/plugin-openapi)
</Section>

<Section title="Usage">
Minimal specification file:

```json
{
  "openapi": "3.1.0",
  "info": {
    "title": "Project API",
    "version": "1.0.0"
  },
  "paths": {
    "/health": {
      "get": {
        "summary": "Check service health",
        "tags": ["system"],
        "responses": {
          "200": {
            "description": "The application responds"
          }
        }
      }
    }
  }
}
```

MDX page example:

```mdx
import { OpenApi } from '@edoc-space/plugin-openapi'

<OpenApi
  source="/storage/edoc/static/openapi.json"
/>
```
</Section>
