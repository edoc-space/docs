# Documentation versions

E-Doc keeps the latest documentation in the regular `docs` tree. Older versions are described in `versions.json` and can override only the files that changed.

The current version always opens at `/docs`. Supported previous versions open at `/docs/v/{version}`. Archived versions stay visible on the "All versions" page, but their documentation tree and search are disabled.

## versions.json

```json
{
  "current": "1.2.0",
  "versions": [
    { "version": "1.2.0", "label": "v1.2.0", "status": "current", "released_at": "2026-07-13" },
    { "version": "1.1.0", "label": "v1.1.0", "status": "supported", "released_at": "2026-07-10" },
    { "version": "1.0.0", "label": "v1.0.0", "status": "supported", "released_at": "2026-07-01" }
  ]
}
```

Statuses:

- `current` - the latest documentation, available at `/docs`.
- `supported` - a previous version, available at `/docs/v/{version}`.
- `archived` - shown as history only; its docs and search are disabled.

The `released_at` field is optional, but recommended: it shows the release date in the version list and changelog.

## since and until

A document or section can define a version window in front matter:

```md
---
title: New feature
since: 1.2.0
---
```

With `since: 1.2.0`, the page is hidden from `1.1.0` and older versions. `until` works in the opposite direction and hides content after the specified version.

If only part of an existing page changed, create an override for the older version and keep the latest text in the regular `docs` tree.
